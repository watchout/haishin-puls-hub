# Haishin+ HUB ペルソナストーリー集

## 0. ドキュメント概要

- プロダクト：Haishin+ HUB（配信プラス HUB）
- 目的：
  - 関係するプレイヤーごとの「理想的なユーザー体験」を物語形式で整理し、
  - 企画・仕様検討・UI設計・営業資料（提案書／LP）に転用できる状態にすること
- 想定共通シナリオ：
  - A社「春の新製品発表セミナー」＠ビジョンセンター丸の内
  - 現地 80 名＋オンライン 200 名
  - 懇親会あり
  - 配信・撮影は配信プラスが担当
  - Haishin+ HUB 上で全プレイヤーが連携している状態

---

## 1. イベント参加者のストーリー

### 1-1. プロフィール（例）

- 氏名：佐藤 彩（32歳）
- 属性：BtoB企業のマーケ担当
- 立場：A社の取引先として招待された参加者
- 参加形態：現地参加

### 1-2. 体験ストーリー

#### ● 事前〜前日

1. A社からの案内メールに、  
   「イベントポータルへのリンク」と「受付用QRコード」が付いている。
2. ポータルを開くと、
   - 開催日時
   - 会場住所と地図（Google Map）
   - タイムテーブル
   - 登壇者情報  
   が一画面でまとまっている。
3. 不安な点をポータル内のAIに質問する：
   - 「服装はスーツじゃないとダメですか？」
   - 「終了は何時頃になりますか？」  
   → 主催が登録した情報を元に AI がわかりやすく回答。

#### ● 当日・受付〜会場まで

1. 会場入口でQRを提示 → 受付タブレットで**ワンタップチェックイン**。  
   紙のリストや名刺は不要。
2. 待機時間にポータルを確認し、
   - Wi-Fi情報
   - 会場フロアマップ
   - トイレ・喫煙所の場所  
   を AI に聞いて把握。
3. 迷ったときも、  
   「懇親会はどこで何時からですか？」とAIに入力すれば即回答。

#### ● 終了後

1. 翌日、「お礼メール」とともに
   - アーカイブ視聴URL
   - 資料DLリンク
   - アンケートフォーム  
   が届く。
2. アンケートはポータル内で数分で回答完了。  
   自由記述も、主催側では AI により「要約＋キーワード抽出」される。

## 2. イベント主催者（セミナー担当）のストーリー

### 2-1. プロフィール（例）

- 氏名：田中 誠（38歳）
- 企業：A社 マーケティング部
- 役割：新製品発表セミナーの企画〜当日運営〜事後報告までの責任者

### 2-2. 体験ストーリー

#### ● 企画〜会場・配信決定

1. Haishin+ HUB にログイン → 「新規イベント作成」ボタンを押す。
2. 入力するのはざっくりした条件だけ：
   - イベント目的：「既存顧客向け新製品発表」
   - ターゲット：「既存取引先の担当者」
   - 想定人数：「現地80・オンライン200」
   - 予算レンジ：「100〜150万円」
   - 日程候補：「3/10〜3/20 の平日」
3. AI が自動で、
   - ビジョンセンター丸の内 ◯◯ ルームの候補日
   - ハイブリッド構成案（配信＋録画＋撮影）
   - 概算見積り
   - 稟議用A4企画書ドラフト  
   を提案してくれる。
4. 田中さんは内容を少し修正し、「稟議用PDF出力」を押して上司に提出。

#### ● 準備〜当日まで

1. イベントごとのダッシュボードには、
   - 「登壇者決定」
   - 「告知開始」
   - 「資料締切」
   - 「リハーサル」  
   など自分が関与するタスクだけが期限付きで並ぶ。
2. 登壇者情報フォームや案内メール文面はAIが生成。  
   田中さんは内容をチェックして送るだけ。
3. HUB のAIに質問：
   - 「まだ資料提出が終わっていない登壇者は？」  
   → 「お二人います。◯◯様・△△様です。リマインドメール案を作成しましょうか？」
4. 「はい」と答えると、丁寧なリマインド文面が下書きに入り、そのまま送信できる。

#### ● 当日〜終了後

1. 当日は Haishin+ HUB のイベント画面を開くだけで、
   - 現地のチェックイン人数
   - オンライン視聴者数
   - 進行状況  
   がリアルタイムで見える。
2. 終了後は何も入力せずとも、
   - 参加者数
   - 質問の傾向
   - アンケート結果
   をまとめた「イベントサマリーレポート」が自動生成される。
3. レポートを添付して上司に送れば、事後報告も完了。

## 3. イベント企画会社（代行会社）のストーリー

### 3-1. プロフィール（例）

- 氏名：中村 遥（35歳）
- 会社：イベント企画・運営代行会社 B社
- 役割：複数クライアントのセミナー・イベントを並行して回すプロデューサー

### 3-2. 体験ストーリー

#### ● クライアント案件の俯瞰管理

1. Haishin+ HUB にログイン → 「代理店ビュー」を開く。
2. A社・C社・D社など、担当している企業ごとに
   - 進行中のイベント
   - ステータス（企画中／準備中／実施済）
   が一覧で表示される。
3. AIに指示：
   - 「今週中に締め切りが来るタスクを、クライアント別に整理して」  
   → 各社ごとの要注意タスク一覧が出る。

#### ● 新規案件の立ち上げ

1. クライアントとのヒアリングメモを、そのまま HUB に貼り付ける。
2. AIに依頼：
   - 「このヒアリング内容から、候補の会場・形式・予算案を3パターン提案して」  
   → 会場候補／配信構成／概算費用を含んだ企画案がまとまって返ってくる。
3. 気に入った案を微修正し、  
   「クライアント提案資料」としてPDF出力して持参。

#### ● 進行管理とクライアント報告

1. 各イベントのタスク進捗は、主催・会場・配信側をまたいで一元管理されている。
2. AIに質問：
   - 「A社の新製品発表セミナーで、遅れているタスクを教えて」  
   → 遅延タスクと担当者の一覧が表示される。
3. AIに依頼：
   - 「それをもとにA社向けの進捗共有メール文面を作って」  
   → そのままコピペしてメール送信できるレベルの文章が出てくる。

## 4. イベント会場運営社（会場チェーン）のストーリー

### 4-1. プロフィール（例）

- 氏名：山田 翔（40歳）
- 会社：ビジョンセンター丸の内 拠点責任者
- 役割：会場予約・当日運営・スタッフ管理

### 4-2. 体験ストーリー

#### ● 拠点全体の見える化

1. HUB にログイン → 「丸の内拠点ダッシュボード」を開く。
2. 今週〜来月までのイベントが、
   - 日時
   - セミナー名
   - 利用形式（会場のみ／ハイブリッド／配信のみ）
   - 想定人数  
   とともに表示される。
3. AIに質問：
   - 「今週の配信付きイベントと、その注意事項をまとめて」  
   → 配信付きイベントだけが抽出され、LAN増設・機材持ち込みなどの特記事項がリストアップされる。

#### ● 現場スタッフのオペレーション

1. 日別ビューに切り替えると、その日のイベント一覧が表示される。
2. 各イベントを開くと、
   - レイアウト図
   - 机・椅子・電源・LAN配置
   - 搬入・搬出時間  
   の指示が確認できる。
3. 現場タブレットからAIに聞く：
   - 「このイベントは標準レイアウトAでいい？変更点ある？」  
   → HUBがイベント情報と主催者の要望を確認し、回答する。

## 5. 動画配信・撮影業者（配信プラス）のストーリー

### 5-1. プロフィール（例）

- 氏名：金子 裕司（42歳）
- 会社：配信プラス（Haishin+）
- 役割：配信・撮影案件の責任者／実務オペレーション

### 5-2. 体験ストーリー

#### ● 週間スケジュール把握

1. HUB にログイン → 「配信プラスビュー」を開く。
2. 自社が担当するイベントだけが、
   - 日時・会場
   - クライアント名
   - 規模（カメラ台数・プラットフォーム）
   - 状態（仕様確定／リハ前／当日／完了）  
   とともに一覧で表示される。
3. AIに聞く：
   - 「来週のイベントのうち、まだ仕様がFIXしていない案件を教えて」  
   → 案件名と、ヒアリング不足項目が表示される。

#### ● 準備〜現場オペ

1. 特定イベントをクリック → 「配信タブ」を開くと、
   - 構成（カメラ／マイク／スイッチャー／回線）
   - 会場の常設機材情報
   - 当日のタイムテーブル（本編／休憩／質疑）  
   がまとめて見える。
2. AIに依頼：
   - 「このイベント用のRunbook（当日マニュアル）を作って」  
   → セットアップ手順／音声・映像チェック項目／進行タイムラインが自動生成される。
3. 前回の類似イベントで起きたトラブルログをもとに、
   - 「前回の◯◯セミナーのトラブルを踏まえて注意点を追記して」  
   とAIに指示すると、Runbookに反映される。

#### ● 主催者とのコミュニケーション

1. AIに質問：
   - 「このイベントで主催側に確認が必要な事項を整理して」  
   → Zoomアカウント提供者・動画再生有無・同時通訳の有無など、未確定項目が一覧化。
2. AIに依頼：
   - 「この内容をベースに確認メール案を作って」  
   → そのまま主催者に送信できる文面が生成される。

#### ● 終了後の提案

1. イベント終了後、HUB の AI が、
   - 当日の内容
   - 参加者の反応
   を踏まえて、
   - ダイジェスト動画制作
   - PR動画制作
   - 多言語版コンテンツ制作  
   などの提案文面の草案を作ってくれる。
2. 金子さんは内容を微修正し、主催者へ「お礼＋提案メール」として送信。

## 6. 登壇者（スピーカー）のストーリー

### 6-1. プロフィール（例）

- 氏名：小林 純（45歳）
- 属性：A社のプロダクトマネージャー
- 役割：新製品プレゼンテーションのメインスピーカー

### 6-2. 体験ストーリー

#### ● 事前準備

1. 「登壇者情報登録のお願い」メールから HUB のフォームを開き、
   - 氏名・肩書き・プロフィール
   - 顔写真
   - 使用予定の資料・動画の有無
   - 当日の登壇方式（現地／オンライン）  
   を入力・アップロード。
2. 送信後、AIが
   - イベントサイト用の紹介文
   - パンフレット用の紹介文  
   を自動生成し、主催者が確認して公開。

#### ● 接続テスト・当日

1. HUB から
   - 接続テスト日時
   - 当日のタイムテーブル（登壇開始時間・持ち時間）
   が自動で共有される。
2. AIに聞く：
   - 「オンライン登壇の時、マイク・カメラ設定はどうすればいい？」  
   → 環境に応じた簡易ガイドが提示される。

## 7. 営業・マーケ（リード活用担当）のストーリー

### 7-1. プロフィール（例）

- 氏名：伊藤 奈央（30歳）
- 企業：A社 営業企画部
- 役割：イベントで獲得したリードのフォロー・商談化

### 7-2. 体験ストーリー

1. イベント終了後、HUB にログイン → 「リード活用ビュー」を開く。
2. 画面には、
   - 参加者一覧（現地／オンライン）
   - アンケート結果
   - 質問内容
   - AIによる「関心度スコア」  
   が表示されている。
3. AIに依頼：
   - 「関心度が高い人だけをピックアップして、営業担当ごとにExcelにエクスポートして」  
   → 営業担当別のリストが作られる。
4. さらに、
   - 「今回のイベントに参加した人向けに、フォローアップメール3パターンを作って」  
   と指示して、MAツール側に流し込む。

## 8. 使用上のメモ

- 本ペルソナストーリーは、
  - プロダクト企画の認識合わせ
  - UI/UX検討
  - 営業資料・提案書・LP のシナリオ
  - 自動SSOT生成エンジンへの入力（ユースケース事例）  
  としての利用を想定。
- 実際のプロジェクトでは、各ペルソナごとに
  - 課題
  - 期待価値
  - 主要KPI  
  を追加定義しても良い。
