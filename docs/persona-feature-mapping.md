# Haishin+ HUB ペルソナ × MVP機能 マッピング表

## 目的

- 各ペルソナが **どのMVP機能から価値を得るか** を一覧化する
- MVP（Phase 1）のスコープ判断・優先度決定の根拠として使う
- SSOT自動生成エンジンへのインプットとして、ペルソナ↔機能↔ペインのトレーサビリティを確保する

## 機能別の詳細解説

### 5.1 イベント企画・予約 AIアシスト

| ペルソナ | 関与度 | 具体的な価値 |
|---|---|---|
| 主催者 | ◎ | 目的・予算・日程を入力 → 会場候補・構成案・企画書ドラフトが自動生成。企画のゼロベース作業を大幅削減。 |
| 企画代行会社 | ◎ | ヒアリングメモから複数パターンの企画案を自動生成。クライアント提案までの工数を短縮。 |
| 会場運営社 | ○ | 自社会場が候補として自動提示される。配信付きパッケージとして営業に活用。 |
| 配信業者 | ○ | 企画段階で配信構成案が組み込まれ、早期の仕様確定につながる。 |

### 5.2 タスク・締め切り一元管理（マスタースケジュール）

| ペルソナ | 関与度 | 具体的な価値 |
|---|---|---|
| 主催者 | ◎ | 全タスクが自動展開され、自分の担当分だけがダッシュボードに表示。抜け漏れ防止。 |
| 企画代行会社 | ◎ | 複数クライアントのタスクを横断的に把握。「代理店ビュー」で優先度管理。 |
| 会場運営社 | ◎ | レイアウト準備・搬入出・設備チェックのタスクが自動アサイン。最新情報が常に1箇所。 |
| 配信業者 | ◎ | 配信・撮影関連タスクが自動展開。リハーサル日程・機材確認の漏れを防止。 |
| 登壇者 | ○ | 資料提出・接続テスト等の締め切りが通知される。主催者との間のステータスが可視化。 |
| 営業マーケ | △ | 事後フォロータスク（リード整理・メール送信）のリマインドとして活用可。 |

### 5.3 ステークホルダー・ワークフロー管理

## MVP スコープ判断への示唆

1. **全ペルソナ共通で最も依存度が高い機能** は `5.2 タスク・締め切り一元管理`（◎が4つ）。  
   → MVP のバックボーン。最優先で実装する。

2. **JV 提案の説得力に直結する機能** は `5.4 参加者レイヤー`。  
   → 会場運営社の「スタッフ負荷軽減」を実感させるデモとして最重要。

3. **売上拡大ストーリーに直結する機能** は `5.1 企画・予約 AIアシスト`。  
   → 会場営業が「配信付きパッケージ」を即提案できるようになるキー機能。

4. **配信プラスの継続収益エンジン** は `5.5 事後レポート＆フォローアップ`。  
   → ダイジェスト動画・多言語展開の自然な提案導線。
