## 5. 価値提案ステートメント

### 5.1 記入欄（メインステートメント）

```
セミナー・イベントの主催者・会場・配信業者・全関係者 が
情報分断・タスク漏れ・属人化した調整業務 を持っているとき、

Haishin+ HUB は
全プレイヤーが1つのシステムに集まり、AIに「聞く・頼む」だけでイベント運営が回る体験 を提供します。

メール・Excel・電話によるバラバラな運営 と違って、
配信業者×会場チェーンのJVだからこそ実現できる「配信込みワンストップ運営OS」 があります。
```

### 5.2 ペルソナ別ステートメント

**主催者向け:**
```
セミナーの企画〜報告まで全部やっている担当者 が
毎回ゼロからの企画書作成・関係者調整・事後レポートに追われているとき、

Haishin+ HUB は
目的・規模を入力するだけでAIが企画書を生成し、タスクが自動展開され、レポートも自動で出る体験 を提供します。
```

**会場向け（JV提案用）:**
```
配信付き案件が増えているのに「箱だけ」の提供に留まっている会場運営者 が
配信の見積り・説明ができず案件が他社に流れているとき、

Haishin+ HUB は
「配信込みワンストップ会場」として即時見積り・パッケージ販売ができ、
受付負荷もAIが軽減する仕組み を提供します。
```

   会場・配信・企画代行・登壇者・営業マーケ・参加者が同じシステムで連携するプロダクトは存在しない。

2. 「配信業者 × 会場チェーンのJV」: EventHub は配信機能を内蔵するが、会場との関係を持たない。
   会場予約システムは会場特化だがイベント運営機能がない。
   配信プラス × ビジョンセンターのJVにより「配信込みワンストップ会場」を実現できるのはこの組み合わせだけ。

3. 「AIファースト設計」: EventHub のブログ・機能ページにAI関連の言及がない（2026年2月時点）。
   既存ツールにAIを後付けするのではなく、「AIに聞く・AIに頼む」を前提としたUI/UX設計。
