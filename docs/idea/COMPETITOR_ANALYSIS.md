## 2. 主要競合の詳細分析

### 2.1 EventHub（最重要競合）

> 出典: [EventHub 機能一覧](https://eventhub.jp/function/)、[EventHub ブログ](https://eventhub.jp/blog/)

#### 概要

| 項目 | 内容 |
|------|------|
| 会社名 | 株式会社EventHub（東京都港区） |
| ポジショニング | 「シェアNo.1 イベントマーケティングプラットフォーム」 |
| ターゲット | 主にBtoB企業のマーケティング部門（イベント主催者） |
| 主な利用シーン | ウェビナー、カンファレンス、展示会、ハイブリッドイベント |
| 価格帯 | 非公開（エンタープライズ向け・個別見積り） |

#### 機能一覧

| カテゴリ | 主な機能 | Haishin+ HUB との比較 |
|---------|---------|----------------------|
| **イベントページ作成** | デザインカスタマイズ、ブランディング反映、登録フォーム、動画視聴ページ、アンケート、多言語対応、独自ドメイン | HUBは参加者ポータルの自動生成。デザインカスタマイズはPhase 1では限定的 |
| **集客・参加者管理** | 登録フォーム、MA/SFA連携（Salesforce, Marketo, HubSpot）、ナーチャリングメール、有料チケット販売、QRコード受付 | HUBはQRチェックインあり。MA連携はPhase 2。チケット販売は対象外 |
| **動画・イベント配信** | ライブ配信、擬似ライブ、アーカイブ、マルチトラック、外部配信連携（Zoom, YouTube, Vimeo） | **HUBは配信機能を持たない**（配信プラスがJVパートナーとして実務担当） |
| **顧客インタラクション** | LIVE掲示板、匿名投稿、リアクション、アンケート、インセンティブ表示 | HUBはAIコンシェルジュが代替。インタラクション機能はPhase 2以降 |
| **分析・アナリティクス** | 分析ダッシュボード、集客・参加統計、リード情報エクスポート、行動ログ・視聴ログ、流入経路分析 | HUBはAIによるサマリーレポート自動生成。詳細視聴ログはPhase 2 |
| **外部ツール連携** | Salesforce, Marketo, HubSpot, Slack, Teams（ノーコード連携） | HUBはPhase 1ではCSVエクスポート。Phase 2でAPI連携 |
| **エンタープライズ** | セキュリティ、管理者権限、カスタマーサポート、SAML認証 | HUBはマルチテナント＋RBAC。SAML認証はPhase 2以降 |
| **出展社・スポンサー** | 出展社管理画面、ブース、メッセージ・商談予約、リード納品、ブース訪問チェックイン | **HUBは対象外**（セミナー・説明会特化） |

#### EventHub のブログから読み取れる注力領域

| 記事テーマ | 示唆 |
|-----------|------|
| 「ウェビナー運営の工数を削減する方法」 | 運営効率化は彼らも課題認識。ただしAI活用は限定的 |
| 「HOTリードを逃さないメール・架電術」 | リード活用・商談化がコア価値。マーケティング寄り |
| 「ウェビナー報告書の作成方法」 | レポート作成の手間は顧客の顕在ペイン。HUBはAI自動生成で差別化可能 |
| 「同一ウェビナーを複数回開催する方法」 | 繰り返し開催の効率化。テンプレート機能の需要あり |

#### EventHub の強み

- **国内シェアNo.1**: ブランド認知・導入実績の厚み
- **MA/SFA連携の成熟度**: Salesforce, Marketo, HubSpot とノーコード連携済み
- **配信機能内蔵**: ライブ・擬似ライブ・アーカイブを自社プラットフォーム内で完結
- **エンタープライズ対応**: SAML認証、ISMS取得（ISO 27001）
- **展示会・カンファレンス対応**: 出展社管理・ブース・商談予約など大規模イベント向け機能

#### EventHub の弱み（Haishin+ HUB から見た攻めどころ）

| 弱み | Haishin+ HUB の優位性 |
|------|----------------------|
| **主催者中心設計**: 会場・配信業者・登壇者など「他プレイヤー」の管理機能がない | 全プレイヤーが同一システムでロール別に稼働 |
| **会場との連携なし**: 会場の空き管理・設備情報・拠点オペレーションに非対応 | JV基盤として会場チェーンの業務を組み込み |
| **AI活用が限定的**: ブログ記事にもAI関連の言及が見当たらない（2026年2月時点） | AIファースト設計。企画書・見積り・メール・レポートの全工程でAI生成 |
| **配信業者視点の不在**: 配信の構成設計・Runbook・追加提案の仕組みがない | 配信プラスのドメイン知識を組み込んだ配信業者ビュー |
| **タスク管理が弱い**: イベント準備のマスタースケジュール・マルチプレイヤータスク管理がない | マスタースケジュールの自動展開＋ロール別ビュー |
| **会場営業ツールではない**: 「配信付きパッケージ」の営業・見積りには使えない | 会場営業が即時見積りを出せるJV基盤 |

### 2.2 Peatix

| 項目 | 内容 |
|------|------|
| ポジショニング | イベント・コミュニティ管理プラットフォーム |
| ターゲット | 主催者（個人〜法人）、集客・チケット販売 |
| 強み | 集客ネットワーク（月間100万人以上のユーザーベース）、使いやすさ、無料プランあり |
| 弱み | 運営管理機能が弱い、配信機能なし、マルチプレイヤー対応なし |
| HUBとの競合度 | **低**: 集客・チケット特化であり、HUBの「運営OS」とは領域が異なる。将来的に集客連携先として検討可 |

### 2.3 EventRegist

| 項目 | 内容 |
|------|------|
| ポジショニング | イベント受付・参加者管理プラットフォーム |
| ターゲット | 法人主催のセミナー・カンファレンス |
| 強み | 受付・参加者管理に特化した安定した機能、国内導入実績 |
| 弱み | EventHubと同様に主催者中心、AI活用なし、配信は外部連携 |
| HUBとの競合度 | **低〜中**: 参加者管理・QR受付で一部機能が重なるが、「運営OS」としてのスコープは大きく異なる |

### 2.4 Bizzabo / Cvent（海外SaaS）

| 項目 | 内容 |
|------|------|
| ポジショニング | エンタープライズ向けイベント管理プラットフォーム（グローバル） |
| ターゲット | 大企業のマーケティング部門（英語圏中心） |
| 強み | 高機能（CRM連携・分析・エンゲージメント）、グローバル導入実績 |
| 弱み | 高価格（年間数百万〜数千万円）、日本語対応不十分、日本の商習慣に合わない |
| HUBとの競合度 | **低**: 価格帯・ターゲット市場が異なる。ただし長期的には注視が必要 |

### 2.5 Excel + メール + Zoom（代替手段）

| 項目 | 内容 |
|------|------|
| ポジショニング | 既存ツールの組み合わせ（＝現状の運営方法） |
| 利用者 | ほぼ全てのセミナー主催者 |
| 強み | 追加コストゼロ、学習コストゼロ、慣れている |
| 弱み | 情報分断、タスク漏れ、属人化、スケールしない |
| HUBとの競合度 | **最も高い**: HUBが置き換えるべき「現状」そのもの。導入障壁＝「今のやり方で十分」という慣性 |

## 5. EventHub に対する競合戦略

### 5.1 正面衝突を避ける領域

| 領域 | EventHub の優位性 | HUBの方針 |
|------|------------------|-----------|
| MA/SFA連携 | Salesforce, Marketo 等とのノーコード連携が成熟 | Phase 2でCSV→APIへ段階的に対応。正面衝突はしない |
| 配信機能 | プラットフォーム内蔵（ライブ・擬似ライブ・アーカイブ） | 配信プラスがJVパートナーとして実務対応。内蔵しない |
| 展示会・大規模カンファレンス | 出展社管理・ブース・商談予約が充実 | 対象外（セミナー・説明会に特化） |
| エンタープライズ機能 | SAML認証、ISMS取得済み | Phase 2以降で必要に応じて対応 |

### 5.2 差別化で攻める領域

| 領域 | EventHub にないもの | HUBの攻め方 |
|------|-------------------|------------|
| **会場JV基盤** | 会場営業・拠点管理・即時見積りに非対応 | 「配信込みワンストップ会場」を実現するJV基盤として、会場チェーン経由で市場を取る |
| **AIファースト** | AI活用の言及がほぼない（2026年2月時点） | 企画書・見積り・メール・レポートの全工程でAI生成。「AIに聞いて・AIに頼む」がUXの軸 |
| **全プレイヤー管理** | 主催者中心。会場・配信・登壇者のロール別管理なし | 全プレイヤーが同一システムでロール別ビューを持つ。情報分断の根本解消 |
| **タスク自動展開** | イベント準備のマスタースケジュール機能なし | イベント種別テンプレートから自動展開。全ロールのタスクが一元管理 |
| **配信業者ビュー** | 配信業者のための管理機能なし | Runbook自動生成、案件管理、追加提案AI生成など配信業者の業務を支援 |

### 5.3 Go-to-Market 戦略の違い

| 観点 | EventHub | Haishin+ HUB |
|------|----------|-------------|
| **販路** | 直販（Webマーケ＋営業チーム） | **JV先（会場チェーン）経由**。会場が顧客に「標準サービス」として提供 |
| **初期ターゲット** | BtoB企業のマーケ部門 | **会場チェーン（ビジョンセンター）＋その会場を使う主催者** |
| **価値訴求** | 「リード獲得・分析の最大化」 | **「全員の手間が減る。AIに頼める」＋「会場の売上が増える」** |
| **導入障壁** | 主催者の意思決定のみ | 会場チェーンとのJV合意が必要（＝一度入ると参入障壁になる） |

## 6. 機会と脅威

### 6.1 機会（Opportunity）

| 機会 | 説明 |
|------|------|
| EventHub のAI空白 | 2026年2月時点でEventHubにAI機能の言及がない。AIファーストで先行できる窓がある |
| 会場チャネルの独占 | 会場チェーンとのJVにより、「この会場で開催するならHUBが標準」という独占チャネルを構築できる |
| レポート自動化の顕在ニーズ | EventHubのブログで「ウェビナー報告書の作成方法」が記事になっている → 顧客の顕在ペイン。HUBはAI自動生成で即解消 |
| 配信業者市場の未開拓 | EventHub含め、配信業者向けの管理機能を持つプラットフォームがない。配信プラス自身のペインを解消しながらプロダクト化 |

### 6.2 脅威（Threat）

| 脅威 | 影響度 | 対策 |
|------|-------|------|
| EventHub がAI機能を追加する | 高 | 早期にJV基盤＋全プレイヤー管理を確立し、AI「だけ」では追いつけない構造を作る |
| EventHub が会場連携を開始する | 中 | ビジョンセンターとの独占JVを先行確立。実運営のドメイン知識で差をつける |
| 新規参入（AIネイティブのイベントSaaS） | 中 | 会場チェーンとのJV関係＋配信業者の実務知識が参入障壁 |
| EventHub の価格攻勢 | 低 | 価格ではなく「会場の売上増＋全員の手間削減」というROIで勝負 |

## 7. 競合分析サマリー

### 7.1 Haishin+ HUB の競合優位性（まとめ）

```
1. 「全プレイヤー横断」:
   EventHub = 主催者向け。HUB = 全員向け。
   これは機能追加では簡単に埋められない「設計思想」の違い。

2. 「AIファースト」:
   EventHub にAI機能なし（2026/2月時点）。
   HUB は全工程にAIを組み込む設計。先行優位を確保する窓がある。

3. 「会場JV基盤」:
   EventHub は会場との関係を持っていない。
   HUB はビジョンセンターとのJVにより「配信込みワンストップ会場」を実現。
   これはパートナーシップに基づく参入障壁。

4. 「配信業者のドメイン知識」:
   配信プラス自身が配信・撮影の現場を知っているからこそ設計できる
   Runbook、追加提案、配信構成の自動化。SaaS企業には持てない知識。
```

### 7.2 注視すべきポイント

| 注視項目 | チェック頻度 | チェック方法 |
|---------|------------|------------|
| EventHub のAI機能追加 | 月次 | 機能ページ・プレスリリース・ブログ確認 |
| EventHub の会場・配信連携の動き | 四半期 | 提携発表・プレスリリース |
| 新規AIネイティブ競合の出現 | 月次 | Product Hunt, TechCrunch Japan, 国内スタートアップDB |
| EventHub の価格戦略変更 | 四半期 | 営業資料・口コミ |
