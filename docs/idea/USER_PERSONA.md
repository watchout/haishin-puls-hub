## ペルソナ 1: 田中 誠（プライマリ — セミナー主催者）

### 基本プロフィール

```
┌─────────────────────────────────────────────────────────────────┐
│  田中 誠 （38歳 / 男性）                                        │
│                                                                 │
│  "セミナーの企画から報告まで全部やるの、正直しんどいです"          │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  職業: マーケティング部 セミナー担当                              │
│  会社: A社（BtoB製造業 / 従業員500名規模）                        │
│  居住地: 東京                                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 詳細情報

| カテゴリ | 内容 |
|---------|------|
| **仕事** | |
| 役職・役割 | マーケティング部 チームリーダー。新製品発表セミナーの企画〜当日運営〜事後報告の全責任者 |
| 主な業務 | 年間10〜20回のセミナー企画・運営。会場手配、配信業者調整、登壇者対応、告知、当日運営、事後レポート |
| 使用ツール | メール、Excel、PowerPoint、Google Calendar、Zoom、各種会場サイト |
| 1日の過ごし方 | メール確認→タスク整理→社内調整会議→資料作成→業者とのやり取り→上長報告 |

### 課題（Pains）

**日々感じている課題**:

| # | 課題 | 深刻度 | 頻度 |
|---|------|-------|------|
| 1 | 企画書を毎回ゼロから作成。過去資料を探して再利用するのも大変 | 高 | 毎回 |
| 2 | 会場・配信・登壇者・関連業者とのやり取りがメール・電話に分散し、最新情報が分からなくなる | 高 | 毎日 |
| 3 | タスクの抜け漏れが怖い。「登壇者の資料まだ来てない」を気付くのが直前になりがち | 高 | 毎週 |
| 4 | 稟議用資料の作成に時間がかかる | 中 | 毎回 |
| 5 | 事後レポート（参加者数・アンケート集計）の作成が面倒 | 中 | 毎回 |

**課題によって失っているもの**:
- 時間: 1イベントあたり延べ40〜100時間（関係者調整含む）
- 機会: 事後データの活用不足で、次回企画・営業連携の質が上がらない
- 精神的負担: 「何か忘れてないか」の不安が常にある

**課題を解決しようとした経験**:

| 試したこと | 結果 | なぜダメだったか |
|-----------|------|----------------|
| Excel管理シートの作成 | 一時的に改善 | ファイルが増え、最新版が分からなくなる |
| チャットツール（Slack）での情報共有 | 情報が流れて埋もれる | ストック情報の管理に向いていない |
| プロジェクト管理ツール（Asana等） | 社外関係者が使えない | 会場・配信・登壇者は別システム |

### ニーズ（Gains）

**求めていること**:

| # | ニーズ | 重要度 |
|---|-------|-------|
| 1 | 企画〜報告までの全工程を1つのシステムで完結させたい | 高 |
| 2 | AIに企画書・メール文面・レポートを作ってもらいたい | 高 |
| 3 | 全関係者のタスク進捗を一画面で把握したい | 高 |
| 4 | 「あれどうなった？」をAIに聞くだけで分かるようにしたい | 中 |

**理想の状態**:
```
ログインして、目的・規模・予算を入力するだけで企画書ドラフトが出てくる。
関係者への連絡もAIが文面を作ってくれる。
当日はダッシュボードを見るだけ。事後レポートは自動生成。
「セミナー担当ってこんなに楽でいいの？」と感じるレベル。
```

### プロダクトとの関係

| 項目 | 内容 |
|------|------|
| どの機能を最も使うか | EVT-001（企画AI）、EVT-010（マスタースケジュール）、EVT-040（レポート自動生成） |
| 利用シーン | イベント企画開始時 → 準備期間中の毎日 → 当日 → 事後 |
| 得られる価値 | 企画〜報告の工数を50%以上削減 |
| 解約リスク | 低（一度使い始めると戻れない） |

### 引用（この人が言いそうなこと）

```
"企画書を毎回ゼロから作るのが本当に面倒。前回のやつ探すのすら大変。"

"登壇者の資料が来てるか来てないか、メール検索しないと分からないのが怖い。"

"セミナー終わった後のレポート作りが地味にキツい。参加者数集計してアンケートまとめて…。"
```

| 3 | 定型質問を受付から減らし、スタッフをイレギュラー対応に集中させたい | 高 |
| 4 | イベント終了後にレポート＋次回提案をセットで送りたい | 中 |

| 項目 | 内容 |
|------|------|
| どの機能を最も使うか | VENUE-001（拠点ダッシュボード）、VENUE-002（即時見積り）、EVT-033（AI FAQ） |
| 利用シーン | 毎朝の拠点確認 → 営業時の見積り生成 → 当日のオペ確認 |
| 得られる価値 | 配信付き案件の受注増 + スタッフ負荷軽減 |
| 解約リスク | 低（JVパートナーとして組み込まれる） |
