## 使い方

1. **棚卸し**: 下記の標準機能カテゴリを見て、必要な機能にチェック
2. **ID付与**: 実装する機能には一意のIDを振る
3. **受入条件**: 各機能に3〜7個の受入条件を設定
4. **状態管理**: 設計中/実装中/完了/スコープ外を明示

### EVT-001: ゴールベースイベント作成 受入条件

```
- [ ] AC1: 「新規イベント作成」画面で、目的/ターゲット/想定参加者数/予算レンジ/日程候補を入力できる
- [ ] AC2: 入力内容をもとにAIが会場候補・形式案・概算見積りを自動生成する
- [ ] AC3: 生成結果を修正・確定し、イベントとして登録できる
- [ ] AC4: 企画代行会社がクライアントに代わってイベント作成できる（代理店ビュー経由）
- [ ] AC5: 過去の類似イベントを参考にした提案が可能（将来改善）
```

### EVT-004: 概算見積りAI自動生成 受入条件

```
- [ ] AC1: 入力された規模・形式に基づき、会場費＋配信・撮影パッケージ費の概算が自動計算される
- [ ] AC2: 標準パッケージ（例: 基本配信/フル配信+撮影/フル配信+撮影+編集）から選択できる
- [ ] AC3: 見積り内容を手動で調整（項目追加・単価修正）できる
- [ ] AC4: 会場営業スタッフが「利用人数・形式・配信有無」だけで即時見積りを出せる
- [ ] AC5: 見積りをPDFまたはメールで共有できる
```

```
- [ ] AC1: イベント情報の登録内容から、参加者向けポータルページが自動生成される
- [ ] AC2: ポータルにはイベント名・日時・タイムテーブル・会場住所＋地図・Wi-Fi情報・登壇者紹介・資料DL・配信URLが含まれる
- [ ] AC3: ポータルはスマートフォンに最適化されたレスポンシブデザインである
- [ ] AC4: ポータルのURLはユニークで、招待メールから1クリックでアクセスできる
- [ ] AC5: イベント情報が更新されると、ポータルにリアルタイムで反映される
- [ ] AC6: イベント終了後もアーカイブとしてアクセス可能（資料DL・アーカイブ視聴）
```

### EVT-033: AIコンシェルジュ（参加者FAQ） 受入条件

```
- [ ] AC1: ポータル上にチャットUIが配置され、参加者が自然言語で質問できる
- [ ] AC2: 会場案内（トイレ・喫煙所・懇親会場所・最寄駅等）に回答できる
- [ ] AC3: イベント情報（開始時間・登壇者・持ち物等）に回答できる
- [ ] AC4: Wi-Fi情報・配信URLなど主催者が登録した情報に基づいて回答する
- [ ] AC5: 回答できない質問には「主催者にお問い合わせください」とエスカレーション導線を表示する
```
