# Haishin+ HUB × 貸し会議室 JV提案ストーリー

## 1. JVの前提整理

### 役割分担

| プレイヤー | 担当領域 |
|---|---|
| **貸し会議室運営事業者（JV先）** | 会場提供・受付・会場オペ。Haishin+ HUB を導入し、顧客との窓口を担う。 |
| **配信プラス（自社）** | 配信・撮影・制作の実務。Haishin+ HUB の企画・運用・改善。専門サポート・標準パッケージの設計。 |

### JV先へのコアメッセージ

1. **売上が増える**
   - 配信・撮影付きプランの販売で「客単価アップ」
   - ハイブリッド対応により「他会場からの乗り換え・新規獲得」
   - リピート率向上（毎回同じ会場でやった方が楽になる）

2. **運営負担が減る**
   - 問い合わせ・見積り・調整・当日案内の手間を、HUB＋AIが肩代わり
   - 現場スタッフの「属人スキル」に依存しない運営

## 2. 場面別ペイン → HUBでの解決

※詳細は `docs/pain-points.md` セクション4 を参照。  
ここではJV提案資料向けに要点を整理する。

### 2-1. 営業・受注フェーズ

- **Before:** 配信は「業者紹介だけ」。見積りに数日かかり、他社に流れる。
- **After:** HUB＋AIで配信込みパッケージ＋概算見積りを即時提示。「配信込みワンストップ会場」として差別化。

### 2-2. 予約確定〜準備フェーズ

- **Before:** メール・電話・Excelがバラバラ。担当変更で引き継ぎ地獄。
- **After:** 「イベントカード」に全情報が一元化。全関係者が同じ情報を見て動く。

### 2-3. 当日現場オペレーション

- **Before:** 受付に質問が集中。配信トラブルで板挟み。
- **After:** 定型質問はポータル＋AI。配信は配信プラスが直接対応。スタッフ負荷が確実に減る。

### 2-4. 終了後〜リピート・アップセル

- **Before:** 「売上計上して終わり」。次回提案の切り口がない。
- **After:** HUBがレポート＋次回提案を自動生成。「イベントの成功体験を振り返るパートナー」に。

## 3. 売上アップモデル（シミュレーション例）

> ※以下の数字は**あくまでシミュレーション例**として使える形にしたもの。

### モデル前提

- 貸し会議室の年間利用件数：1,000件
- うち、セミナー・説明会など「配信と相性が良い利用」：300件

### Before（導入前のよくある状態）

| 項目 | 数値 |
|---|---|
| 配信付き案件化率 | 300件中 10%（30件） |
| 1件あたり配信関連売上 | 紹介料ベースで 1〜2万円/件 |
| 会場側の配信関連売上 | **年間 約60万円** |

### After（Haishin+ HUB＋JV後のイメージ）

| 項目 | 数値 |
|---|---|
| HUB上で「配信付き標準パッケージ」を商品化 | 営業が提案しやすくなる |
| 配信付き案件化率 | 300件中 30〜50%（90〜150件）を狙う |
| 1件あたり配信＋撮影パッケージ | 15〜30万円 |
| うち会場側取り分（JVレベニューシェア） | 3〜5万円/件 |

### 売上イメージ

| 指標 | 金額 |
|---|---|
| 配信付き 100件 × 会場取り分 4万円 | **年間 400万円** |
| 導入前との差分 | **+340万円/年のアップサイド** |

> ※実際は客単価・配信構成・レベニューシェア率により変動  
> ※重要なのは「箱だけ」→「配信込み会場」として単価を引き上げる構造

## 4. 運用効率アップのイメージ

### Before（よくある状態）

- **営業・予約担当：** 見積り・配信可否・空き状況確認に1件あたり数十分〜1時間
- **現場スタッフ：** 案内・トイレ・Wi-Fiなどの質問対応で常にバタバタ

### After（HUB導入後）

- **営業：** 見積り・配信パッケージ提案をHUB＋AIが支援 → 1案件あたりの対応時間を半分以下に
- **現場：** 定型質問はポータル＋AIに集約 → スタッフはイレギュラー対応に集中 → 人員計画も立てやすくなる

### 数値例

| 指標 | Before | After | 効果 |
|---|---|---|---|
| 1件あたり営業対応時間 | 60分 | 30分 | 年間1,000件なら **約500時間削減** |
| 当日スタッフ質問対応時間 | − | − | 体感 **2〜3割削減** を目指せる設計 |

## 5. JV先向け提案資料の構成案

以下のスライド構成で、ビジョンセンター等への提案資料を作成可能。

| # | スライドタイトル | 内容 |
|---|---|---|
| 1 | タイトル | 「配信プラス × 貸し会議室 JV 企画案」<br>サブ：Haishin+ HUB による会場価値向上プラン |
| 2 | 会場業界の変化 | ハイブリッド需要の増加。「配信に強い会場」へのシフト。 |
| 3 | 貸し会議室運営者のペイン | 営業・受注 / 準備調整 / 当日運営 / 事後フォロー・リピート |
| 4 | Haishin+ HUB のコンセプト | 「イベントの全プレイヤーが集まるハブ」<br>「画面を探すのではなく、AIに聞いて・AIに頼む」 |
| 5 | ペイン → HUB での解決対応表 | セクション2の表をスライド化 |
| 6 | 売上アップモデル | Before/After の案件数と売上モデル（シミュレーション） |
| 7 | 運用効率アップのイメージ | 営業工数削減 / 現場負荷軽減 |
| 8 | JVの役割分担と収益分配イメージ | 会場：会場＋配信をワンストップで販売<br>配信プラス：配信・撮影・システム運用 |
| 9 | 次のステップ | パイロット会場の選定 → 3〜6ヶ月のトライアル → KPI設定 |

### 提案時の重点KPI候補

- 配信付き案件化率（Before → After）
- リピート率（同一会場での再利用率）
- 営業1件あたりの対応時間
- クレーム・トラブル件数の削減率
- 会場側の配信関連売上（月次・年次）
